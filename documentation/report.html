1. Abstract

The AI Travel Planner Smart Guide is a full-stack web application that leverages NVIDIA's NIM API to generate personalized, day-wise travel itineraries. The system features a React.js frontend styled with Tailwind CSS, a Flask REST API backend, and MongoDB for data persistence. Users can input their budget, travel preferences, duration, and interests, after which the AI generates a comprehensive itinerary with detailed activity schedules, cost breakdowns, meal suggestions, accommodation recommendations, and safety tips. The platform also includes real-time weather information via OpenWeatherMap API, a smart recommendation engine, user reviews, an admin dashboard with analytics, and PDF export capabilities.

Keywords: Artificial Intelligence Travel Planning NVIDIA NIM API Full-Stack Web App React.js Flask Llama 3.1 405B

2. Introduction

2.1 Background

Travel planning is a complex process that requires consideration of multiple factors — budget constraints, interests, safety, weather conditions, local attractions, and transportation logistics. Traditional travel planning either relies on expensive travel agents or time-consuming manual research across multiple platforms.

2.2 Problem Statement

Existing travel planning tools lack intelligent personalization and require significant manual effort. There is a need for an automated system that can understand individual traveler preferences and generate comprehensive, budget-aware itineraries with minimal user effort.

2.3 Proposed Solution

The AI Travel Planner Smart Guide solves this by integrating NVIDIA's NIM API with a rich destination database, real-time weather data, and user reviews to deliver personalized, actionable travel plans within seconds.

3. Objectives

  1. Develop a modern, responsive web application for intelligent travel planning
  2. Integrate NVIDIA NIM API for automated itinerary generation with structured JSON output
  3. Implement JWT-based authentication with admin and user roles
  4. Design an optimized MongoDB database with GridFS for image storage
  5. Build a recommendation engine using tag-based and budget similarity matching
  6. Integrate real-time weather data with smart travel advice
  7. Provide PDF export functionality for generated itineraries
  8. Create an admin dashboard with analytics on searches, popular tags, and budget trends
  9. Ensure production-level security with bcrypt hashing, input validation, and CORS protection

7. Module Description

🔐 Authentication Module

JWT-based auth with bcrypt password hashing. Supports user registration, login, profile management, and role-based access (user/admin).

🗺️ Destination Management

Full CRUD for destinations with multi-image upload via GridFS. Supports text search, tag filtering, and budget-based queries.

🤖 AI Itinerary Engine

Integrates with NVIDIA NIM API to generate structured day-wise itineraries with cost breakdowns, hotel recommendations, and safety tips.

🌤️ Weather Module

Fetches real-time weather via OpenWeatherMap API with smart travel advice based on temperature, humidity, and conditions.

⭐ Reviews System

Users rate and review destinations. Supports image upload. Admins can moderate (delete) inappropriate reviews.

🎯 Recommendation Engine

Suggests similar destinations using tag overlap and budget range similarity. Provides personalized recommendations based on user interests.

📊 Analytics Dashboard

Admin analytics showing most searched destinations, popular travel types, average budget trends, and platform statistics.

📄 PDF Export

Server-side PDF generation using ReportLab. Produces formatted itinerary documents with activities, costs, meals, and tips.

8. AI Integration (NVIDIA NIM)

8.1 Prompt Engineering Strategy

The system uses structured prompt engineering to ensure consistent, parseable JSON responses from NVIDIA NIM AI:

  1. Context Injection: Destination data from MongoDB is included in the prompt (attractions, hotels, tips, tags)
  2. User Preferences: Budget, travel type, duration, interests, transport/stay/food preferences
  3. Strict JSON Schema: The prompt specifies the exact JSON structure Gemini must return
  4. Fallback Handling: Response cleaning removes markdown code fences before JSON parsing

9. API Design

All endpoints are prefixed with /api. Protected routes require a Bearer JWT token.

Authentication Endpoints

Method Endpoint Auth Description
POST /api/auth/register — Register new user
POST /api/auth/login — Login, returns JWT
GET /api/auth/profile User Get current user profile
PUT /api/auth/profile User Update profile

Destination Endpoints

Method Endpoint Auth Description
GET /api/destinations/ — List with search, filter, pagination
GET /api/destinations/all — All destinations (dropdown)
GET /api/destinations/:id — Get single destination
POST /api/destinations/ Admin Create with images
PUT /api/destinations/:id Admin Update destination
DELETE /api/destinations/:id Admin Delete destination
GET /api/destinations/:id/similar — Get recommendations
GET /api/destinations/image/:id — Retrieve GridFS image

Itinerary Endpoints

13. Installation & Setup Guide

Prerequisites

  • Node.js 18+ & npm
  • Python 3.10+
  • MongoDB 6+ (running locally on port 27017)
  • Google Gemini API key
  • OpenWeatherMap API key

Step 1: Backend Setup

cd backend
pip install -r requirements.txt

# Edit .env with your API keys:
# GEMINI_API_KEY=your-key-here
# WEATHER_API_KEY=your-key-here

python app.py
# Server starts at http://localhost:5000

Step 2: Frontend Setup

cd frontend
npm install
npm run dev
# App opens at http://localhost:5173

Step 3: Default Admin Login

Email: admin@travelplanner.com
Password: admin123

Environment Variables (.env)

Variable Description
GEMINI_API_KEY Google Gemini API key
WEATHER_API_KEY OpenWeatherMap API key
MONGO_URI MongoDB connection string
SECRET_KEY Flask secret key
JWT_SECRET_KEY JWT signing secret
